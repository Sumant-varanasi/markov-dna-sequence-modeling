# Markov Chain DNA Sequence Modeling

[![Python](https://img.shields.io/badge/Python-3.8%2B-brightgreen.svg)](https://www.python.org)  
[![License](https://img.shields.io/badge/License-MIT-yellow.svg)](LICENSE)

A project exploring the application of **Markov Chains** to model genomic DNA sequences and generate statistically realistic synthetic DNA.  
Trained on the *Escherichia coli* K‑12 genome, the model captures nucleotide dependencies, k‑mer frequencies, entropy, and GC content – preserving both global and local sequence patterns.

**Key Highlight**  
Trinucleotide (order‑3) Markov Chains provide the optimal balance between **predictive accuracy** and **computational efficiency**, faithfully reproducing real‑world genomic statistics.

## 🎯 Features

- **N‑gram Markov Chain training** on real DNA (1 ≤ n ≤ 4)
- **Synthetic sequence generation** of any length
- **Statistical comparison metrics:**
  - Base composition (A, C, G, T)
  - K‑mer entropy
  - KL divergence (real vs. synthetic k‑mer distributions)
  - Prediction accuracy
  - Motif counts (e.g., ATG, TATA, GCG)
  - Sliding‑window GC content analysis
- **Visualizations:** heatmaps, bar charts, GC curves, accuracy/entropy plots
- **Automatic NCBI download** if local genome file is missing

## 📖 How It Works

1. The real DNA sequence is cleaned (only A, C, G, T retained).
2. A Markov chain of order `n` is built: states are `n`‑grams, and transition probabilities are estimated from observed counts.
3. A synthetic sequence is generated by repeatedly sampling the next nucleotide from the current state’s probability distribution.
4. Statistical metrics compare the real and synthetic sequences to validate the model.

The full mathematical framework and results are detailed in the [project report](project_report.pdf).
